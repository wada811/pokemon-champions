# 📋 月次情報収集 運用方針

このドキュメントは、ポケモンチャンピオンズに関連する信頼できる情報源の維持・管理と、
月次情報収集の運用ルールを定めています。

---

## 目的

- ポケモンチャンピオンズ固有の情報源と、比較・検証に使う競技ポケモンの参考情報源をリポジトリ内で一元管理する
- 月次の定期確認により、情報源の最新性・有効性を保つ
- アプリのデータ（タイプチャート等）が現実のゲーム仕様と乖離しないよう監視する
- 月次レポートや関連ドキュメントを日本語中心の自然な表現で管理する

---

## 月次収集の流れ

毎月1日に GitHub Actions が自動的に収集用 Issue を作成します。

```
毎月1日 0:00 UTC (= JST 9:00)
    ↓
GitHub Actions が月次 Issue を自動作成
    ↓
担当者が docs/sources.md の情報源を確認
    ↓
変更あり → PR を作成してファイル更新
変更なし → Issue にサマリーをコメントして Close
    ↓
docs/collection-results/YYYY-MM.md に結果を記録
```

### 収集対象の分類

#### 定期確認しやすい情報源

- ポケモン公式サイトのお知らせ一覧
- ポケモンチャンピオンズ公式サイトの商品・更新情報
- ポケモンずかんの掲載内容

#### 人手確認が必要

- 国内向け公式サイトの更新情報・新発表
- タイプチャートや技・特性の仕様変更
- 日本語圏で確認できる大会結果やチームトレンド

### 記録時の文言ルール

- 月次収集では、日本語で確認できる情報源のみを原則として使用する
- 収集結果の本文は日本語を基本とし、簡単な英単語やサービス名のみ必要最小限で使う
- ポケモン名・大会名・フォルム名は、可能な限り日本語の公式表記に合わせる
- 日本語圏の公開情報だけでは断定できない内容は、推測せず「確認できず」と明記する

---

## 変更が必要なケース

以下の条件を満たす場合、リポジトリのファイルを更新してください。

| 条件 | 対象ファイル |
|------|-------------|
| タイプチャートに変更があった | `src/data/typeChart.ts` |
| 新ポケモン・新フォルムが追加された | `src/data/typeChart.ts`、README |
| 技・特性の仕様変更があった | 該当するデータファイル |
| 情報源の URL・運営状況に変化があった | `docs/sources.md` |
| README のメタ情報・機能説明が古くなった | `README.md` |

---

## 変更不要なケース（記録のみ）

変更が不要な月でも、以下を必ず残してください。

1. Issue に収集サマリーをコメント（環境トップポケモン・注目トレンド・次回確認事項）
2. `docs/collection-results/YYYY-MM.md` を作成して結果を記録
3. Issue を Close する

---

## 情報源の見直しルール

詳細は [docs/sources.md](./sources.md#見直しルール) を参照してください。

月次収集中に情報源の閉鎖・長期停止・信頼性低下を確認した場合は、
`docs/sources.md` の更新 PR を同月中に作成してください。
